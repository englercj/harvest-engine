
## Includes

All includes should use quotes, except for system includes (such as STL). Including files from contrib packages brought in by the package manager should be quoted.

Includes should be grouped by where they are coming from. Within each group they should be ordered alphabetically. The definition, and order, of the groups is:

- Header for this implementation (if .cpp)
- Local includes in your module implementation (src/)
- Module includes, including the public includes of your own module (include/)
- Contrib includes
- System includes

For example:

```cpp
// Example `allocator.cpp` file

// Copyright Chad Engler

#include "he/core/allocator.h"

#include "allocator_helpers.h"

#include "he/core/assert.h"
#include "he/core/compiler.h"
#include "he/core/config.h"
#include "he/core/utils.h"

#include "fmt/core.h"

#include <new>
```

## Getters

Getters should use the name of what they get without any prefix.

Example:

```cpp
class Example
{
public:
    uint32_t Value() const { return m_value; }

private:
    uint32_t m_value;
}
```

## Setters

Setters should use the name of what they set with a `Set` prefix.

Example:

```cpp
class Example
{
public:
    void SetValue(uint32_t value) { m_value = value; }

private:
    uint32_t m_value;
}
```

## Macros

### Naming

Macros names should be in UPPER_SNAKE_CASE and be prefixed with `HE_`. Macro parameters should be lowerCamelCase.

Example:

```cpp
#define HE_EXAMPLE(paramOne, paramTwo) paramOne ## paramTwo
```

### Fully Qualified Names

All generated code, including code generated by macros, should use fully qualified names. This includes the `::` prefix.

Macros can be used in any namespace context, so always using the fully qualified name for types ensures the macro works in any context.

Example:

```cpp
#define HE_MAKE_STRING(name, value) ::he::String name(#value)
```
